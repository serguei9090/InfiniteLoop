Project Rules: Flutter Atomic Design (Hybrid)

You are an expert Flutter Engineer. You are working in a project generated by very_good_cli but refactored to use Atomic Design.

1. Folder Structure & Organization

You must strictly follow this directory structure. Do not place files in random locations.

lib/ui/: EXCLUSIVE home for all visual widgets.

atoms/: Smallest indivisible units (Buttons, Text, Icons, InputFields). No business logic allowed.

molecules/: Combinations of atoms (SearchBars, UserListTiles).

organisms/: Complex sections (Headers, ProductGrids, Forms).

templates/: Page layouts (scaffolding without data).

pages/: Full screens. This is the ONLY place where BlocProvider, BlocBuilder, or Cubit logic is injected.

lib/logic/: Feature-based business logic (Cubits, Blocs, Repositories).

Example: lib/logic/auth/auth_cubit.dart.

2. Atomic Design Principles

Atoms are "Dumb": They take data via final properties in the constructor. They never access APIs or Blocs directly.

Reuse First: Never write Text("Hello", style: TextStyle(...)) inside a page. Use AppText.header("Hello") from atoms/app_text.dart.

Composition: Molecules should be built only by importing Atoms. Organisms should be built by importing Molecules and Atoms.

3. Coding Standards (Very Good Analysis)

You must enforce strict Dart linting rules:

CONST: Always add const to constructors and widgets where possible.

TYPES: Never use var or dynamic unless absolutely necessary. Explicitly type variables (e.g., final int count, not final count).

KEYS: Every widget constructor must have a super.key.

COMMAS: Always add a trailing comma , to the last parameter of a function or list.

IMPORTS: Prefer absolute imports (e.g., import 'package:my_app/ui/atoms/atoms.dart';) over relative imports.

4. Testing Strategy

Mirror the Structure: If you create lib/ui/atoms/my_button.dart, you must create test/ui/atoms/my_button_test.dart.

PumpWidgets: When testing Atoms, strictly test their visual properties (color, text) and tap callbacks.

Bloc Tests: When testing Logic, use bloc_test to verify state changes.

5. Error Prevention

If you modify a file, ensure you do not break the closing brackets })));.

If a compilation error occurs regarding "missing const", fix it immediately.

If a widget becomes too large (>150 lines), refactor it into smaller Atoms or Molecules.
