# 🛠️ Orchestrator Tool Schema and Usage Guidelines

This document serves as a single source of truth for data formats and tool interaction expectations within the autonomous agent workflow. All modules must adhere to these schemas when passing structured information or requesting context from external tools.

## 1. Structured Output Format (Surgical Edits)
When the Orchestrator receives code modifications, they MUST be in a parsable JSON/Dictionary format, NOT free-form text diffs.

**Schema:**
```json
{
  "files": [
    {
      "file_path": "src/module_name.py",
      "changes": [
        {
          "line": 12,
          "old_content": "def old_function():",
          "new_content": "async def new_service_function(user):"
        },
        {
          "line": 45,
          "old_content": "return status:",
          "new_content": "return (status, result);"
        }
      ]
    }
  ],
  "summary": "Brief description of the changes applied."
}
```

## 2. Resource Monitoring Payload (PSUtil Integration)
Resource spikes must trigger a specific internal event payload that signals the need for context cleanup and model unloading. This is an internal signal, not user input.

**Payload Example:**
`{"event_type": "VRAM_HIGH", "usage_percent": 92.5, "threshold": 90}`

## 3. Task Persistence Schema (State Saving)
The state of the automation loop must be saved in a predictable JSON format to allow resumption.

**Schema:**
```json
{
  "last_task_id": "T-42",
  "current_status": "awaiting_model_response",
  "history_length": 50,
  "failed_retries": 0,
  "next_attempt_at": "YYYY-MM-DDTHH:MM:SSZ"
}
```

## 4. General Tool Usage Reminder
*   **Code Search:** Use `Grep` for content searching (`pattern="regex"`), and `Glob` for file path discovery (`pattern="**/*.py"`). Never use raw bash `grep`.
*   **API Interaction:** All external service calls (Ollama, Git) must be wrapped in dedicated `asyncio.Task`s with robust `try...except` blocks to ensure the main loop does not crash on transient network or API errors.
